## How it works

You tell Claude Code what to build. tstack decides how.

**One command. The right scope.**

`/tstack-task` — fix, rename, tweak. Minutes. Single commit.

`/tstack-feature` — build a flow, add a page, connect systems. Hours. Opens a PR.

`/tstack-mission` — build an entire system. Days. Multiple PRs, tracked state.

Every tier runs the same quality gate: `lint → build → test`. No broken code ships.

---

## Pure markdown. Zero dependencies.

tstack is 23 slash commands, 2 agents, 8 skills, and 5 hooks — all plain `.md` files symlinked to `~/.claude/`.

```bash
git clone https://github.com/tstack-framework/tai.git ~/tstack && ~/tstack/setup
```

That's it. No packages. No runtime. No config files.

---

## What you get

- **Structured tiers** — stop guessing whether this is a task or a feature
- **Quality gate** — lint, build, test after every change. No opt-out.
- **Agent coordination** — features spawn the right agents and merge their work
- **Session continuity** — `/tstack-resume` picks up where you left off
- **Missions** — multi-day work tracked with ROADMAP.md and state.json
- **Extensions** — add project-specific commands, agents, and skills
